# Plaid Auto-Import Setup (Phase 2)

This document walks you through enabling automatic card-linked expense import via Plaid.

## What you need

- A Plaid developer account (free at [plaid.com/docs/quickstart](https://plaid.com/docs/quickstart/))
- Your `client_id` and `secret` from the Plaid Dashboard
- Vercel deployment (already set up)

## Step 1 — Create your Plaid account

1. Go to [dashboard.plaid.com](https://dashboard.plaid.com)
2. Sign up for a free developer account
3. In **Team Settings → Keys**, copy your `client_id` and the `Sandbox secret`
4. When ready for real bank connections, copy the `Development` or `Production` secret instead

## Step 2 — Add environment variables to Vercel

In your Vercel project dashboard → **Settings → Environment Variables**, add:

| Name | Value |
|------|-------|
| `PLAID_CLIENT_ID` | Your Plaid client_id |
| `PLAID_SECRET` | Your Plaid sandbox/production secret |
| `PLAID_ENV` | `sandbox` (or `development` / `production`) |

## Step 3 — Install the Plaid SDK

```bash
npm install plaid
```

Then commit `package.json` and redeploy.

## Step 4 — Activate the serverless function

In `api/plaid-link-token.js`, uncomment the implementation block (lines are clearly marked with comments).

You'll also need a second endpoint to exchange the public token for an access token after the user links their card — scaffold it as `api/plaid-exchange-token.js` following the same pattern.

## Step 5 — Update PlaidView.jsx

Replace the placeholder UI in `src/PlaidView.jsx` with the Plaid Link button:

```jsx
import { usePlaidLink } from "react-plaid-link";

// 1. Fetch link token from /api/plaid-link-token on mount
// 2. Pass it to usePlaidLink({ token, onSuccess })
// 3. In onSuccess, POST public_token to /api/plaid-exchange-token
// 4. Store the access_token in your DB (encrypted) per user
// 5. Poll /api/plaid-transactions to sync new charges into the expenses table
```

Install the React SDK: `npm install react-plaid-link`

## Step 6 — Transaction sync

Create `api/plaid-sync-transactions.js` to:
1. Accept a `userId` and look up their stored `access_token`
2. Call `client.transactionsSync(...)` 
3. Map each transaction to an expense row (use the `category` field from Plaid + your own mapping)
4. Upsert into the `expenses` Supabase table (use transaction ID as idempotency key to avoid duplicates)

## Security notes

- Never store the Plaid `access_token` in the frontend or in plain text
- Use Supabase's encrypted columns or a secrets manager for access tokens
- Plaid requires an annual security review for Production access — Sandbox and Development are unrestricted

## Cost

Plaid's pricing depends on the product tier. For the Transactions product:
- **Sandbox**: free, unlimited test data
- **Development**: free up to 100 Items (linked accounts)  
- **Production**: billed per Item/month — check [plaid.com/pricing](https://plaid.com/pricing) for current rates
